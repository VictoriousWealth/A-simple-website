Accessibility statement

This accessibility statement applies to this website name and only. This website is run by the author, name at the end of the page. This website strives to be accessible to as many as it can, meaning you should be able to:

  • zoom in up to 150% without the text spilling off the screen;
  • listen to the website using a screen reader (including NonVisual Desktop Access, Microsoft Narrator (for Microsoft and Windows users), Google TalkBack (app version existent for android devices) and Window-Eyes).

I have also ensured that any of the website's text and images are as simple as possible to understand.
AbilityNet has advice on making your device easier to use if you have a disability.

  1. Introduction The ShefFamily's website is committed to ensuring digital accessibility for people with disabilities. I am continually improving user experience and striving to adhere to the relevant accessibility standards.

  2. Accessibility FeaturesI have implemented the following features:
    1. Alternative text for all images, providing information about the content and function of images, helping users who rely on screen readers.
    2. Semantic HTML, i.e. the website uses different types of headings to structure content, different types of lists
    3. Keyboard accessibility, ensuring that all interactive elements can be accessed and operated using the keyboard alone. I did this due to knowing that it is crucial for users who may not be able to use a mouse.
    4. Focus indication, so users who rely on screen readers, can understand where they are on the page by focusing on links which change color when they are in focus
    5. Form accessibility to an extent, as in, all the forms on the website include clear labels on each field. However, no error messages are produced.
    6. Text contrast against its background, making it easier to read for users with vision impairement.
    7. Resizable text without any loss of content or functionality up to 280px in width screen.
    8. Responsive design, meaning, adapting to different screen sizes, benefiting users who may be using a variety of devices.
    9. Consistent navigation throughout the website, aiding users in understanding and navigating the site.

  3. Assistive Technologies CompatibilityMy website is designed to be compatible with some commonly used assistive technologies, i.e. screen readers and keyboard navigation.

  4. Known Accessibility IssuesAs of now, after testing the website numerous times, no issues have been encountered.

  5. Contact InformationIf you encounter any accessibility barriers on my website or have feedback on how I can improve, please let me know by contacting me via email: NEOni1@sheffield.ac.uk .

  6. Monitoring and EnforcementAs of now (end-date tbc), I shall regularly assess the website to identify and rectify any accessibility issues. Anyways, I encourage users to report any problems they might encounter.

  7. Further AssistanceIf you require assistance or have specific accessibility questions, please contact me via this email: NEOni1@sheffield.ac.uk.

  8. Review and UpdatesThis accessibility statement will be reviewed and updated periodically starting from [date tbc] to reflect any changes in my commitment or improvements made to the website's accessibility.
